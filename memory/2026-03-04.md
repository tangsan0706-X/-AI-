## 🐛 解决的问题

### 问题 1：输出节点没有下载按钮
**现象：** 节点显示"已完成"但看不到下载按钮

**原因分析：**
1. 输出节点数据存储在 `node.output.media` 字段
2. 显示逻辑检查的是 `node.output.video`
3. 条件判断失败，按钮不显示

**解决方案：**
```javascript
// 增强的数据提取函数
function getOutputData(type) {
  // 1. 直接访问 (output.video)
  if (output[type]) return output[type]

  // 2. Media 对象访问 (output.media.video)
  if (output.media && output.media[type]) return output.media[type]

  // 3. Media 字符串访问 (output.media 直接是 URL)
  if (type === 'video' && typeof output.media === 'string') return output.media

  // 4. 智能搜索所有字段中的 URL
  // ...
}
```

**文件位置：**
- `src/components/Canvas/NodeConfigPanel.vue:getOutputData()`

### 问题 2：工作流执行失败
**错误日志：**
```
节点 #2 (AI文生图) 执行失败: Error: API 请求失败 (500)
节点 #3 (AI图生视频) 执行失败: Error: 输入图片不能为空
节点 #4 (输出) 执行结果: {media: undefined}
```

**根本原因：** 加速器规则模式导致 API 请求被代理

**诊断过程：**
1. 检查 DNS 解析 → 发现被劫持到 `198.18.0.125`
2. 检查 API Key → 格式正确
3. 检查网络环境 → 用户使用加速器规则模式
4. 测试代理影响 → 关闭加速器后成功

**解决方案：**
- 方案 1：临时关闭加速器
- 方案 2：配置直连规则 `*.aliyuncs.com,DIRECT`
- 方案 3：更换 DNS 服务器

### 问题 3：刷新页面后执行结果丢失
**现象：** 工作流执行成功后刷新页面，节点输出数据丢失

### DashScope API 调用流程

```
1. 提交异步任务
   POST /api/v1/services/aigc/text2image/image-synthesis
   Headers: X-DashScope-Async: enable
   Response: { output: { task_id: "xxx" } }

2. 轮询任务状态
   GET /api/v1/tasks/{task_id}
   Response: { output: { task_status: "RUNNING/SUCCEEDED/FAILED" } }

3. 获取结果
   Response: { output: { results: [{ url: "https://..." }] } }
```

### 画布节点执行流程

```javascript
runPipeline() {
  // 1. 拓扑排序获取执行顺序
  const order = getTopoOrder()

  // 2. 收集每个节点的输出
  const outputs = {}

  // 3. 按顺序执行节点
  for (const nodeId of order) {
    // 收集上游输入
    const inputData = {}
    edges.filter(e => e.targetNode === nodeId).forEach(e => {
      inputData[e.targetPort] = outputs[e.sourceNode]?.[e.sourcePort]
    })

    // 执行节点
    const result = await executeNode(node, inputData)

    // 保存输出
    outputs[nodeId] = result
    node.output = result
  }

  // 4. 自动保存
  saveToLocal()
}
```

## 🎓 学到的经验

### 1. 网络环境诊断很重要
- 不要假设用户的网络环境是完美的
- DNS 劫持、代理、防火墙都可能导致问题
- 提供诊断工具比直接说"检查网络"更有帮助

### 2. 数据结构要灵活处理
- API 返回的数据结构可能不一致
- 节点之间的数据传递要考虑多种情况
- 使用智能函数而不是硬编码字段名

### 3. 用户体验细节
- 自动保存比手动保存更友好
- 状态指示器（绿点/红点）比文字更直观
- 错误提示要具体，不要只说"失败"

### 4. 文档很重要
- 即使是小项目也要写 README
- 常见问题要有配置指南
- 每日记录帮助回顾和接续工作
